# Сборка Mac-приложения (DMG) — просто, по шагам

Идея: одна команда превращает твой код в `Plashker-Installer.dmg` — файл-образ,
который любой человек на Mac открывает и перетаскивает приложение в «Программы».
Собирать нужно **на Mac** (образ для Mac делается только на Mac).

Я уже всё подготовил в проекте:
- `Plashker-mac.spec` — конфиг сборки приложения (иконки, привязка `.plshk`)
- `AppIcon.icns`, `DocIcon.icns` — иконки приложения и документа
- `plashker_dmg_background_1320x800.png` — фон окна DMG
- `build-mac.sh` — скрипт, который делает всё сам

---

## Шаг 0. Разовая подготовка (только в первый раз)

Открой **Terminal** (Spotlight → напиши `Terminal` → Enter).

1) Проверь Python 3:
```bash
python3 --version
```
Если версии нет — Mac предложит поставить «инструменты командной строки», согласись.

2) Поставь `create-dmg` — он делает красивый DMG с фоном. Нужен **Homebrew**:
```bash
brew install create-dmg
```
Нет Homebrew? Поставь его одной строкой с https://brew.sh, потом повтори команду.
> Если не хочешь возиться с Homebrew — можно пропустить. Скрипт тогда сделает
> DMG попроще (без фона), но полностью рабочий.

---

## Шаг 1. Собрать DMG (одна команда)

Зайди в папку проекта и запусти скрипт:
```bash
cd "/Users/igor_morozov/Documents/FUN/PLASH_KER"
chmod +x build-mac.sh
./build-mac.sh
```

Что произойдёт (3–6 минут):
1. Создастся окружение и поставятся зависимости.
2. PyInstaller соберёт `dist/Plashker.app`.
3. Из него сделается `Plashker-Installer.dmg` в корне проекта.

В конце увидишь строку `Установщик: …/Plashker-Installer.dmg` — **это он.**

---

## Шаг 2. Проверить и раздать

Дважды кликни `Plashker-Installer.dmg` → откроется окно → перетащи **Plashker**
в **Программы** → запусти из Launchpad или папки «Программы».

**Важно про первый запуск (у тебя и у всех, кому отдашь).** Приложение не
подписано платным сертификатом Apple, поэтому при первом открытии macOS скажет
«Не удаётся проверить разработчика». Это нормально для своей сборки. Открыть так:
**правый клик по Plashker → «Открыть» → в диалоге ещё раз «Открыть»**. Дальше
приложение запускается обычным двойным кликом.
> Совсем убрать это окно можно только платной подписью (Apple Developer, 99$/год).
> Для раздачи коллегам/друзьям не обязательно.

Просто запусти скрипт снова — он пересоберёт с нуля:
```bash
cd "/Users/igor_morozov/Documents/FUN/PLASH_KER"

```

## Если что-то пошло не так

**`pyinstaller: command not found`** — окружение не активировалось. Скрипт это
делает сам; если запускал команды вручную — сначала `source .venv/bin/activate`.

**Белое пустое окно при запуске** — на этом Mac проблема с WebKit-движком;
перезапусти приложение, обычно проходит. Если нет — напиши мне.

**Скрипт ругается на архитектуру у другого человека** — ты собрал под свой
процессор (Apple Silicon M1/M2/M3 или Intel). Собранное на Apple Silicon
запускается и на Intel через Rosetta, но не наоборот. Если нужен универсальный
вариант — скажи, настрою.
