# Plashker

Автоматическая раскладка рекламных «плашек» (баннеров) фильма по множеству
форматов. Дизайнер готовит **по одному PNG максимального разрешения на каждый
элемент** (тайтл, дата, рейтинг, РЕКЛАМА, легал), а приложение само
расставляет их по всем форматам и экспортирует.

Это **первая версия** — движок раскладки и экспорта реализован и проверен
end-to-end, UI на HTML/CSS/JS работает в нативном окне через pywebview.

---

## Технологический стек (зафиксирован спецификацией)

- **Python 3.12** + **Pillow** — движок композиции (вся геометрия и рендер).
- **HTML / CSS / JS** в нативном окне через **pywebview** (без Electron / Node).
- **numpy** — быстрый расчёт bbox для safe-zone по инвертированной альфе.

Логика приложения целиком в Python; JS отвечает только за интерфейс и общается
с движком через мост `Api` (см. `plashker/app.py`).

## Запуск

```bash
cd plashker

# 1. демо-ассеты и конфиги (создаёт data/ и examples/MICHAEL/)
#    уже сгенерированы и лежат в репозитории, шаг нужен только если их удалить:
python tools/make_demo_assets.py

# 2. окружение
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. запуск приложения
python run.py
```

Откроется приветственный экран → «Открыть демо» загрузит проект `MICHAEL`.

### Linux

`pywebview` использует системный webview. Нужен один из бэкендов:

```bash
# GTK:
sudo apt install python3-gi gir1.2-webkit2-4.1
# или Qt:
pip install pyqt6 pyqt6-webengine
```

В окружении без графики (CI, контейнер) GUI не поднимется — это нормально,
движок при этом полностью работоспособен (см. ниже).

### Проверка движка без GUI

```bash
# отрисовать один формат или все сразу в renders/
python tools/render_one.py --all
python tools/render_one.py 16x9 --safe        # с подсветкой safe-zone
```

## Архитектура

```
plashker/
  engine/
    models.py       — dataclasses: форматы, элементы, настройки, safe-zone
    assets.py       — загрузка PNG, кэш альфы, content_bbox (обычная и
                      инвертированная — для safe-zone)
    geometry.py     — проценты↔пиксели, деление safe-zone на четверти,
                      масштаб «по занимаемому проценту», клампы
    compositor.py   — сборка кадра: тайтл → дата → рейтинг → легал,
                      опциональные оверлеи фона / safe-zone
  project.py        — .plshk (zip) и папка-проект, рендер, экспорт,
                      имена файлов и структура папок
  app.py            — мост Api (методы, которые дёргает UI), нативные диалоги
ui/                 — index.html / styles.css / app.js (4-зонный интерфейс)
data/               — конфиги форматов, safe-zone PNG, глобальные ассеты
examples/MICHAEL/   — демо-проект
tools/              — генератор демо-ассетов, CLI-рендер
run.py              — точка входа (создаёт окно pywebview)
```

### Ключевые принципы (из спецификации)

- Вся геометрия — в **процентах** холста, не в пикселях.
- Конфигурация форматов и регионов — **данные** (JSON), не хардкод.
- Масштаб элемента — это **процент занимаемого места** (видимый контент
  занимает N% ширины или высоты по своей оси привязки), а не множитель.
  Движок сам считает коэффициент; масштабирование только пропорциональное.
- **Неразрушающий** авто-кроп по альфе: bbox контента кэшируется, файл не режется.
- **Safe Zone** — один залитый PNG на формат с прозрачной «дыркой» = безопасной
  зоной; рамка считается по **инвертированной** альфе.
- Safe-zone делится на 4 четверти: тайтл → верх-лево, рейтинг → верх-право,
  РЕКЛАМА → низ-лево, наш/платформенный легал → низ-право
  (для BYYD 320×480 нижние четверти меняются местами — `legal_mirrored`).
- Дата не в четвертях: центрируется по середине тайтла, под ним с отступом.
- Флаги предпросмотра (`show_background`, `show_safe_zone`) **никогда** не
  влияют на экспорт PNG/PSD.

### Экспорт

Имена: `{movie_title}_{region}_plashka_{w}x{h}-{status}` (для RU регион
опускается). Структура — 4 соц-формата (16x9, 9x16, 4x5, 1x1) с подпапками
регионов + плоская папка `byyd/`. JPEG-на-согласование рендерится **отдельно,
с фоном**.

## Допущения и расширения относительно спецификации

1. **Слайдеры масштаба легала.** В тексте (стр. 31) упомянуты слайдеры
   масштаба для элементов легала, но в схеме манифеста они отсутствовали.
   Добавлены поля `ad_label_scale_pct` / `our_legal_scale_pct` /
   `platform_legal_scale_pct` со значениями по умолчанию.
2. **Деление safe-zone на четверти** — это трактовка автора реализации,
   которую спецификация явно помечает как требующую подтверждения. Реализовано
   буквально и изолировано в `geometry.split_quarters` — легко изменить.
3. **DA-формат `da_1280x720`** экспортируется в папку `da/`. «5 папок» из
   спецификации покрывают стартовый набор (соц + BYYD); DA — ситуативный.
4. Округления рейтинга KZ (88/118 px) — по таблице из спецификации.

## Не реализовано в v1

- **PSD-экспорт.** Спецификация (стр. 14) требует его как обязательный, но сама
  же помечает как самый рискованный компонент, требующий отдельного
  прототипа в первый же день. В `export()` сейчас PNG + опциональный JPEG;
  точка расширения под PSD помечена в коде. Нужно выбрать библиотеку
  (`pytoshop` / `psd-tools` запись / ручная сборка) — это отдельная задача.
- Полноценный undo (сейчас — только в рамках сессии).

### PSD: про встроенные смарт-объекты (п.7)

Настоящие встроенные смарт-объекты Photoshop — это не «слой-картинка», а
связка из (а) linked-layer блоков `lnkD`/`lnk2`, где лежат байты исходного
файла, и (б) per-layer дескрипторов `PlLd`/`SoLd` с матрицей размещения
(8 double-координат), сериализованных в бинарном формате Adobe Descriptor
(OSType-ключи `Objc`/`doub`/`TEXT`/`VlLs`…). Это реализуемо в принципе, но
здесь сознательно не делается «вслепую»: любую ошибку в дескрипторе нельзя
проверить без самого Photoshop, а в офлайн-окружении нет и библиотеки записи
смарт-объектов (`psd-tools` пишет их ограниченно и недоступен без сети) —
риск выдать файл, который Photoshop откажется открывать, выше пользы.
Поэтому writer остаётся надёжным послойным растровым PSD (открывается в любом
редакторе), а исходники для смарт-объектов поставляются рядом в `_originals/`.

## История версий (ранее)

0.5.0 — пятая итерация:

- **Отмена экспорта** в системном диалоге больше не приводит к ложному
  «успешно» — экспорт не запускается.
- **Индикатор площади ВО (п.2):** показывает % площади кадра и порог 5%
  (метка на ползунке, зелёный/красный статус) — по закону ВО иногда обязана
  занимать не менее 5%.
- **Окно настроек шире** (1080px), всё видно без скролла; превью сейф-зон
  рисуются чётким контуром, а не блёклой полупрозрачной заливкой.
- **Соло-режим карточки (п.4):** кнопка в левом верхнем углу изолирует один
  формат; экспорт одной картинки кладёт файлы без лишних подпапок.
- **Точки-перфорация над лентой убраны** — больше места под превью.
- **Юр.инфо площадки — выше основной** (раньше была ниже).
- **«Заменить…»** с троеточием; подменю — устойчивый поповер (клик
  открывает/закрывает, есть «мостик» под курсор, крупная зона клика).
- **Окно экспорта:** порядок PNG → JPEG → PSD; «без юр.информации»
  отделён от переключателя.
- **PSD-экспорт (п.10):** послойный PSD из 4 слоёв (Тайтл/Дата/ВО/Юр.блок)
  + сведённое превью. Слои растровые; в Photoshop превращаются в смарт-объект
  одним ПКМ. Встроенные смарт-объекты на запись пока не делаем — это
  отдельный риск-компонент (см. ниже).

> PSD: смарт-объекты. Сейчас writer пишет корректный многослойный PSD без
> сжатия каналов. Настоящие встроенные смарт-объекты (linked-layer
> дескрипторы) — следующий, более рискованный шаг; намеренно не включены,
> чтобы не получить файл, который Photoshop откажется открывать.

- **Исправлены Safe Zone.** Рамки форматов считались как «весь холст»
